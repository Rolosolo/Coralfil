# Security Policy

## Reporting a Vulnerability

We take security seriously at Coralfil Inc. If you discover a security vulnerability in this project, please report it responsibly to ensure the safety of our users and systems.

### How to Report

**Please do not open a public GitHub issue for security vulnerabilities.**

Instead, email your vulnerability report to: **hello@coralfil.com** with the subject line: **"Bug Bounty"**

Include the following information in your report:

- Description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact or severity
- Suggested fix (if you have one)
- Your contact information and preferred method of communication

### Response Timeline

We commit to:

- **Acknowledge** receipt of your report within 48 hours
- **Assess** the vulnerability and determine severity within 5 business days
- **Develop** and test a fix for confirmed vulnerabilities
- **Release** a security patch as soon as possible (typically within 30 days for critical issues)
- **Notify** you when a patch is released and credit you (if desired) in the release notes

### Supported Versions

| Version | Supported          |
|---------|-------------------|
| 3.x     | ✅ Supported      |
| < 4.0   | ❌ Not supported  |

Only the latest version receives security updates. We recommend upgrading to the latest release as soon as possible.

### Security Best Practices

When using this project, please follow these best practices:

- Keep your installation updated to the latest version
- Review dependencies regularly for known vulnerabilities
- Use environment variables for sensitive configuration (API keys, credentials)
- Never commit secrets to the repository
- Use strong authentication mechanisms in your implementation
- Follow the principle of least privilege for access control
- Enable two-factor authentication for GitHub accounts with repository access

### Responsible Disclosure

We support responsible disclosure and will:

- Not pursue legal action against researchers who report vulnerabilities in good faith
- Provide reasonable time for vulnerability remediation before public disclosure
- Credit security researchers (unless they prefer to remain anonymous)
- Keep your information confidential during the remediation process

Please allow us adequate time to address the vulnerability before disclosing it publicly. We ask for a minimum of 90 days notice, but may coordinate a different timeline based on severity and complexity.

## Data Policy

### Data Collection and Usage

Coralfil Inc. respects user privacy and commits to transparent data handling practices.

#### What Data We Collect

**Usage Data:**
- Anonymous usage statistics (features used, error logs)
- System information necessary for debugging and optimization
- Performance metrics and diagnostic data

**User-Provided Data:**
- Information you voluntarily provide through documentation or support channels
- Configuration data needed for system operation

#### What We Do NOT Collect

- Personal information without explicit consent
- Biometric data
- Precise geolocation data
- Payment information (handled by third-party processors with their own privacy policies)
- Browsing history or session recordings

### Data Usage

We use collected data for:

- Improving product functionality and performance
- Debugging issues and providing technical support
- Security monitoring and vulnerability detection
- Compliance with legal obligations
- Research and development (only with anonymized data)

### Data Retention

- Usage and diagnostic data: Retained for 90 days, then deleted
- Error logs: Retained for 30 days unless needed for active investigations
- Support correspondence: Retained for 2 years for reference and compliance
- Account data: Deleted upon request or account termination, within 30 days

### Data Protection

We implement industry-standard security measures to protect your data:

- Encryption in transit (TLS/SSL for all data transfers)
- Encryption at rest for sensitive data
- Access controls and authentication
- Regular security audits and vulnerability assessments
- Employee data protection training
- Incident response procedures

### Third-Party Data Sharing

We do not sell, trade, or share your personal data with third parties except:

- Service providers with data processing agreements (hosting, analytics, monitoring)
- Legal requirements (law enforcement, court orders)
- Merger or acquisition scenarios (with notice and choice)

### Data Subject Rights

You have the right to:

- Access data we hold about you
- Request correction of inaccurate data
- Request deletion of your data (subject to legal/technical constraints)
- Opt-out of non-essential data collection
- Data portability in structured, machine-readable formats

To exercise these rights, contact: **hello@coralfil.com**

### Cookies and Tracking

This repository and associated documentation sites may use:

- **Essential cookies** for authentication and functionality
- **Analytics cookies** (optional, with consent) to understand usage patterns
- **Third-party tracking** only with explicit user consent

Users can disable non-essential cookies in their browser settings without affecting core functionality.

### Data Breaches

In the event of a data breach that may affect your information:

- We will notify affected individuals within 72 hours
- We will provide details of what occurred and steps being taken
- We will offer remediation resources and support
- We will cooperate with relevant authorities

### Policy Changes

We may update this policy periodically. We will:

- Notify users of material changes via email or prominent notice
- Provide at least 30 days notice before changes take effect
- Allow users to object to non-material changes

The current version and effective date are always available in this repository.

### Compliance

This policy complies with:

- **GDPR** (General Data Protection Regulation)
- **CCPA** (California Consumer Privacy Act)
- **PIPEDA** (Personal Information Protection and Electronic Documents Act)
- **Industry security standards** (OWASP, NIST guidelines)
